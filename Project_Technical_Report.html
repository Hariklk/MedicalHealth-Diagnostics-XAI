Medical Diagnostics AI Project
Technical & Clinical Report

1. Executive Summary

The Medical Diagnostics AI is a comprehensive full-stack and machine learning web platform designed to analyze patient clinical data and predict the risk of various health conditions, primarily focusing on Heart Health and Heart Failure. It leverages state-of-the-art predictive modeling and connects patients with medical professionals. It also features a built-in AI chatbot capable of analyzing symptoms and giving general health guidance.

2. System Architecture

The application runs on a robust modern technology stack structured for rapid inference and high availability:

  • Backend Environment: Node.js with Express.js REST API providing unified routing.
  • Database: MySQL relational database managing User Profiles (Patients and Professionals) and storing prediction histories.
  • Machine Learning Engine: Python (utilizing scikit-learn and pandas) integrated via child_process spawn to handle live diagnostic inferences.
  • Large Language Model (LLM) Integration: Groq API utilizing the llama-3.3-70b-versatile model to provide real-time, conversational medical guidance (Chatbot).

3. Machine Learning Implementations

The core analytical engines are powered by ensemble learning models designed to prevent overfitting and provide robust clinical predictions.

Heart Disease Prediction Engine

  • Algorithm: Random Forest Classifier utilizing hyperparameter optimization.
  • Optimization: Utilizes a CalibratedClassifierCV (sigmoid method) to fix probability overconfidence, allowing the system to output a reliable percentage score of risk (0-100%).
  • Risk Categorization: Low (<40%), Moderate (40%-70%), High (>70%).
  • Explainability: Calculates real-time Feature Importance to inform the user which specific health indicators contributed most significantly to the prediction.

Heart Failure Prediction Engine

  • Algorithm: Random Forest Classifier with balanced class weights to handle asymmetric patient data and prevent bias toward healthy predictions.
  • Data Processing: Evaluates complex multi-dimensional medical data mapped dynamically to encoded numerical formats utilizing LabelEncoder.

4. Glossary of Technical & Clinical Terms

The following terms represent the core parameters evaluated by the AI models during diagnostic processing:

System & Algorithmic Terminology

  • Random Forest Classifier: An ensemble learning method for classification that operates by constructing a multitude of decision trees at training time and outputting the mode of the classes.
  • Model Calibration: A post-processing technique to ensure that a model's predicted probabilities align with the actual observed probabilities, critical for accurate patient risk percentages.
  • Feature Importance: A score given to different diagnostic inputs that indicates how useful or valuable each feature was in the construction of the predictive trees.
  • Class Weight Balancing: A technique used in training the algorithm to pay more attention to the minority class (e.g., severe heart failure) so the model maintains high recall.

Clinical Diagnostic Features

  • Age & Sex: Basic demographic indicators heavily weighted in cardiovascular risk calculations.
  • CP (Chest Pain Type): Categorized into Typical Angina, Atypical Angina, Non-anginal pain, or Asymptomatic.
  • Trestbps / RestingBP: Resting blood pressure measured in mm Hg on admission to the hospital.
  • Chol (Cholesterol): Serum cholesterol measured in mg/dl.
  • FBS / FastingBS (Fasting Blood Sugar): Indicates if fasting blood sugar > 120 mg/dl (1 = true; 0 = false).
  • RestECG: Resting electrocardiographic results (Normal, ST-T wave abnormality, or left ventricular hypertrophy).
  • Thalach / MaxHR: Maximum heart rate achieved during a stress test.
  • Exang / Exercise Angina: Indicates whether chest pain or discomfort was induced by physical exercise.
  • Oldpeak: ST depression induced by exercise relative to rest. Corresponds to an abnormal electrocardiogram indicating acute stress.
  • ST_Slope: The slope of the peak exercise ST segment (Up-sloping, Flat, Down-sloping).
  • CA: Number of major blood vessels (0-3) colored by fluoroscopy.
  • Thal (Thalassemia): A blood disorder (categorized typically as Normal, Fixed Defect, Reversable Defect).
  • Extended Synthetic Indicators: Additional self-reported lifestyle features tracked by the model, including Swelling (Legs/Belly), Rapid Heartbeat, Wheezing, Difficulty Concentrating, Smoking status, and Drinking Patterns.
